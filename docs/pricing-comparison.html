Recommendation

One Supabase Pro project (~₹2,100/mo) covers your ENTIRE operation — it's multi-tenant, so the same project serves every shop and all 2,400+ customers. That cost does not multiply per shop or per card.

If you want it cheaper (your ~₹500/mo target): self-host Supabase/Postgres on a small VPS for ₹400–600/mo. Same code, no lock-in (it's Postgres), you just manage backups/updates. Good move once you have a few paying shops.

Avoid Firebase here: per-operation billing is unpredictable at this read volume and NoSQL fights the ledger/invoice/FIFO logic Khatape is built on.

Estimates for planning only — actual costs depend on usage. Khatape's data is small (money/ledger rows), so cost is dominated by the flat plan, not transaction volume.
